Ασκηση 1η - Ανακοινώθηκε την 24/03/2008 - μετράει για 15% του τελικού βαθμού

Φτιάξτε έργο που προσομοιώνει την ηλεκτρική εγκατάσταση ενός σπιτιού που έχει τους εξής τύπους συσκευών: Θερμοσίφωνο, Φωτιστικό, Κλιματιστικό. Κάθε συσκευή πρέπει να έχει τις ιδιότητες: κατανάλωση (πόσα βατ ηλεκτρικού ρεύματος καταναλώνει), και είναιΑναμμένη. Η κατανάλωση των θερμοσίφωνων είναι σταθερή από κατασκευής τους. Αντίθετα, στα φωτιστικά η κατανάλωση μπορεί να αλλάξει (ενόσω είναι σβηστά, βέβαια), πχ όταν τους αλλάξουμε λαμπτήρα. Επίσης, τα κλιματιστικά μπορούν να λειτουργούν σε πλήρη ισχύ, ή σε χαμηλή ισχύ που είναι η μισή της μέγιστης ισχύος.
Ο κώδικας που θα φτιάξετε θα πρέπει να ελέγχει και να επιβάλει 'λογική' κατάσταση στα στιγμιότυπα, πχ δεν μπορεί να υπάρχει φωτιστικό με κατανάλωση -20W, ή δεν μπορεί να αλλάξει η κατανάλωση του θερμοσίφωνα - μόνο να προβληθεί.
Απαράβατος όρος: σωστή οδόντωση και σχόλια.
Αφού γράψετε τις τρεις κλάσεις, κατασκευάστε αντικείμενα για ένα σπίτι που έχει τρία φωτιστικά, ένα θερμοσίφωνο και δύο κλιματιστικά και αναβοσβήστε τα.

(α) Φτιάξτε διάγραμμα κλάσης που να απεικονίζει τις κλάσεις μαζί με τις ιδιότητές τους και τις μεθόδους τους.
(β) Φτιάξτε διάγραμμα αντικειμένων για το συγκεκριμένο σπίτι.
(γ) Φτιάξτε κλάση που κατασκευάζει τις συγκεκριμένες ηλεκτρικές συσκευές και ανάβει μερικές από αυτές. Φτιάξτε μια μέθοδο που τυπώνει την κατάσταση για κάθε συσκευή και την τρέχουσα κατανάλωση για όλες τις συσκευές.

Τα (α)-(γ) θα τα παραδώσετε ηλεκτρονικά στον υπεύθυνο του εργαστηρίου σας μέχρι την Τετάρτη 2/4 και θα επιδείξετε τα (α)-(β) σε έντυπη μορφή στο εργαστήριο της Πέμπτης 3/4, όπου θα κληθείτε να τα επεκτείνετε.
Προσοχή: μόνον όσοι έχουν παραδώσει την άσκηση θα μπορούν να συμμετάσχουν στην εξέταση και ως εκ τούτου θα βαθμολογηθούν.

Ασκηση 2η - Ανακοινώθηκε την 08/04/2008 - μετράει για 20% του τελικού βαθμού

Χρησιμοποιήστε τον κώδικα του έργου auction ως σημείο εκκίνησης για να κάνετε τις παρακάτω επεκτάσεις:
Ε1: Διατηρείτε τους ανθρώπους που συμμετέχουν στον πλειστηριασμό σε μια συλλογή ως ιδιότητα της Auction. Θα χρειαστεί να προσθέσετε και μια μέθοδο enterPerson στην κλάση Auction.
Ε2: Τροποποιήστε το πρόγραμμα, ώστε να μπορεί να τεθεί τιμή εκκίνησης (ιδιότητα minBid) για κάθε αγαθό. Μια προσφορά γίνεται αποδεκτή, μόνον αν υπερβαίνει την τιμή εκκίνησης. Βέβαια, επιτρέπεται κάποια αγαθά να μην έχουν τιμή εκκίνησης.
Ε3: Προσθέστε μέθοδο public void close() στην κλάση Auction που θα εκτελείται στον τερματισμό του πλειστηριασμού. Αυτή θα διαπερνά τη συλλογή και θα τυπώνει λεπτομέρειες για τα αγαθά. Για κάθε αγαθό που έχει γίνει αποδεκτή προσφορά, τυπώνεται το όνομα του πλειοδότη και η αξία της προσφοράς.
Ε4: Φτιάξτε μέθοδο bidSuccessfully(String pname) που επιστρέφει τις επιτυχημένες προσφορές (αγαθό, αξία) που έχει υποβάλει άνθρωπος με συγκεκριμένο όνομα μέχρι στιγμής.
Ε5: Φτιάξτε μέθοδο που τυπώνει για κάθε άνθρωπο που είχε συμμετοχή στον πλειστηριασμο τα στοιχεία του, πόσες προσφορές υπέβαλε (επιτυχείς, αποτυχημένες, σύνολο), τις επιτυχείς (αυτές που πλειοδότησαν σε αγαθό) και πόσα χρήματα πρέπει να καταθέσει γι αυτές.
Ε6: Φτιάξτε κλάση AuctionDemo που υλοποιεί μια απλή διεπαφή χρήστη ρυθμού διαταγών (command line user interface). Μέσω αυτής μπορείτε να διεξάγετε έναν πλειστηριασμό, καλώντας τις κατάλληλες μεθόδους και περνώντας τους τις απαιτούμενες παραμέτρους.

Τονίζεται η σημασία της επίσημης τεκμηρίωσης (τύπου javadoc). Θα παραδώσετε ηλεκτρονικά μέχρι την Τρίτη 6/5. Στο εργαστήριο της 8/5 θα δείξετε την άσκησή σας. Κατόπιν θα κληθείτε να την επεκτείνετε.

Ασκηση 3η - Επιλέγετε μια από τις δύο εναλλακτικές - μετράει για 15% του τελικού βαθμού

Εναλλακτική α - Ανακοινώθηκε την 13/05/2008

Φτιάξτε μια κομψή, εύχρηστη και εποπτική γραφική διεπαφή χρήστη (graphical user interface) για την εφαρμογή του πλειστηριασμού. Αρχικά θα σχεδιάσετε το παρουσιαστικό της και θα περιγράψετε τη λειτουργικότητά της. Επειτα θα την υλοποιήσετε σε Java. Το βασικό σας μέλημα θα είναι η φιλικότητα προς τον χρήστη: αν δώσετε σε κάποιον το πρόγραμμά σας να τρέχει, θα πρέπει να μπορεί να το χρησιμοποιήσει για να αναπαράγει την εξέλιξη ενός εικονικού πλειστηριασμού και να δει τα αποτελέσματά του, χωρίς παραπάνω οδηγίες και βοήθεια. Η διεπαφή δεν θα πρέπει να έχει αναδυόμενα παράθυρα (πχ showInputDialog ή showMessageDialog της JOptionPane), γιατί τα περιεχόμενά τους εξαφανίζονται όταν πατήσεις OK.
Θα διατηρήσετε εντελώς χωριστά τον κώδικα της διεπαφής (στην κλάση AuctionGUI), από τον κώδικα που υλοποιεί τη λειτουργικότητα του πλειστηριασμού, δηλαδή τις κλάσεις Person, Lot, Bid και Auction. Λογικά, δεν χρειάζεται να πειράξετε τον κώδικα που έχετε φτιάξει από την άσκηση 2. Απλώς θα καλέσετε τις μεθόδους του από τους χειριστές γεγονότων των εξαρτημάτων.

Εναλλακτική β - Ανακοινώθηκε την 26/5/2008

Φτιάξτε κλάση που διαχειρίζεται χρηματικά ποσά. Ένα χρηματικό ποσό αποτελείται από συγκεκριμένες ποσότητες νομισμάτων. Φτιάξτε δύο κατασκευαστές (θα εγείρουν εξαίρεση IllegalArgumentException, αν κάποια παράμετρος είναι παράνομη, πχ αρνητική), εκτυπωτικό, μέθοδο ypologAxia που υπολογίζει την αξία, μέθοδο elaxNomismata που επιστρέφει ισόποσο χρηματικό ποσό με το ελάχιστο πλήθος νομισμάτων.

Για λόγους απλούστευσης, θεωρήστε ότι υπάρχουν νομίσματα των 50, 20, 10, 5, 2, 1 λεπτών (μόνο κέρματα - όχι χαρτονομίσματα). Για να το δοκιμάσετε, βάλτε στην main:
Poson r = new Poson (0.81);
System.out.println(r.toString()); // -> 1 πενηνταλεπτο 1 εικοσαλεπτο 1 δεκαλεπτο 1 λεπτο
Poson p = new Poson (1, 0, 2, 1, 3, 0);
System.out.println(p.toString()); // -> 1 πενηνταλεπτο 2 δεκαλεπτα 1 πενταλεπτο 3 διλεπτα
System.out.println(p.ypologAxia()); // -> 0.78
Poson q = p.elaxNomismata();
System.out.println(q.toString()); // -> 1 πενηνταλεπτο 1 εικοσαλεπτο 1 πενταλεπτο 1 διλεπτο 1 λεπτο
Poson s = new Poson (1,2,3,4,5,-6); // -> java.lang.IllegalArgumentException: Περίμενα μη αρνητικό αριθμό

Θα παραδώσετε ηλεκτρονικά μέχρι την Τρίτη 3/6. Στο εργαστήριο της 5/6 θα δείξετε την άσκησή σας. Κατόπιν θα κληθείτε να την επεκτείνετε.

Εναλλακτικές ασκήσεις: Μπορούν να αντικαταστήσουν οποιαδήποτε από τις προγραμματισμένες ασκήσεις του εξαμήνου με αντίστοιχο ποσοστό. Πριν να ξεκινήσετε, συνεννοηθείτε με τον διδάσκοντα για να σας γίνει ανάθεση και να σας δοθούν διευκρινήσεις.

Ασκηση Εναλλακτική 2 - Ανακοινώθηκε την 25/2/2008 - μετράει για 30-45% του τελικού βαθμού (μπορεί να σας απαλλάξει από όλες τις ασκήσεις).

Επιλέξτε ένα πρόγραμμα που να περιλαμβάνει γραφική απεικόνιση και φτιάξτε κατάλληλο σενάριο για το Greenfoot. Ως ενδεικτικά παραδείγματα αναφέρονται: μια μηχανή έκδοσης εισιτηρίων, η προσομοίωση μιας οδικής διασταύρωσης, η τοποθέτηση οχημάτων στο γκαράζ πλοίου με πολλούς προορισμούς. Για τους ενδιαφερόμενους θα γίνει ειδική παρουσίαση των προγραμματιστικών δυνατοτήτων του Greenfoot και πρόσθετη βοήθεια κατά την υλοποίηση.
